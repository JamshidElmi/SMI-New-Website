

# SMI Home — Component-based structure

This project is split into independent components, each in its own folder.
Edit one file, the change shows up everywhere — no copy-paste.

```
SMI/
├── SMI Home.html              # Thin shell with placeholders (the "source of truth")
├── SMI Home.baked.html        # Pre-baked output — opens directly from file://
├── build.py                   # Bakes the shell + components into the .baked.html
├── css/                       # Global styles shared by all components
│   ├── base.css               # Variables, reset, .wrap
│   ├── buttons.css            # .btn, .btn.light
│   ├── sections.css           # .section, .section-head, headings
│   └── responsive.css         # Global media queries
├── js/
│   ├── component-loader.js    # Optional: fetches components at runtime
│   └── main.js                # Sticky-header scroll behavior
└── components/                # One folder per page section
    ├── header/        index.html + style.css
    ├── hero/          index.html + style.css
    ├── what-we-do/    index.html + style.css
    ├── what-we-do-cards/    style.css (variant)
    ├── what-we-do-tabs/     style.css (variant)
    ├── what-we-do-feature/  style.css (variant)
    ├── ebike-programs/  index.html + style.css
    ├── projects/      index.html + style.css
    ├── partners/      index.html + style.css
    ├── team/          index.html + style.css
    ├── history/       index.html + style.css
    ├── contact/       index.html + style.css
    └── footer/        index.html + style.css
```

## Two ways to view the page

### A) Baked file (no server needed) — recommended for quick viewing

A pre-baked copy is generated by `build.py`. Just double-click
`SMI Home.baked.html` to open it in your browser. No server, no fetch.

### B) Local dev server (live editing)

`js/component-loader.js` uses `fetch()` to load each component. Browsers
block `fetch()` on `file://`, so serve the folder over http:

```bash
python -m http.server 8000        # then open http://localhost:8000
# or
npx serve .                       # Node
php -S localhost:8000             # PHP
```

VS Code users: install **Live Server**, right-click `SMI Home.html` →
"Open with Live Server".

## Editing workflow

1. Edit `components/<name>/index.html` and/or `style.css`.
2. If you're using the baked file: re-run `python build.py` then refresh
   the browser. If you're using the dev server: just refresh.

## Switching the "What We Do" style

Three variants are available in addition to the original:

- `components/what-we-do/`         — original 5-up cards
- `components/what-we-do-cards/`   — bordered cards with hover lift
- `components/what-we-do-tabs/`    — dark editorial numbered list
- `components/what-we-do-feature/` — alternating zigzag feature rows

To swap, change the `data-component` value in `SMI Home.html` to the
desired folder name, e.g.:

```html
<div data-component="what-we-do-tabs"></div>
```

Then re-run `python build.py` (or just refresh if using the dev server).

## Switching the "Projects" style

Four variants are available in addition to the original:

- `components/projects/`        — original 2×2 dark cards
- `components/projects-grid/`   — 2×2 image tiles with overlay text + hover zoom
- `components/projects-list/`   — editorial list rows with side thumbnail + arrow
- `components/projects-mosaic/` — bento asymmetric grid (1 big + 3 small)
- `components/projects-spread/` — full-width magazine spreads, alternating sides

To swap:

```html
<div data-component="projects-list"></div>
```

Then re-run `python build.py`.
